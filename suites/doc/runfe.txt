 A.  ENVIRONMENT VARIABLES/FLAGS:

     ING_TST     This variable points to the top of your testing area
                 structure.  This variable must be set in order to
                 execute this script.  This variable should be set in
                 your environment setup file.


     TST_OUTPUT  This variable will point to the location of the 
                 output files generated by running this script. This
                 variable is set in the script to point to:

                 UNIX: $TST_OUTPUT/fe/local.
                 WINDOWS: %TST_OUTPUT%\fe\local

     TST_INIT	 This variable points to the location of the 
                 initialization files.
	
     TST_CFG     This variable points to the location of the .cfg 
                 files.

                 UNIX: $ING_TST/suites/acceptst
                 WINDOWS: %ING_TST%\suites\acceptst

     TST_SHELL   This variable points to the location of the shell files.

                 UNIX: $ING_TST/suites/shell
                 WINDOWS: %ING_TST%\suites\bat

     SEPPARAM_STAR  This variable indicates which sep test will run 
                 when executing against a STAR database.  If set to
                 false, all parts of the sep test are run. If set to
                 true, the db proc parts of the sep tests are skipped.

***NOTE***

	UNIX

	To verify your environment variables are correct see sample script in
	$ING_TST/suites/userenv/tstenv

	WINDOWS

	To verify your environment variables are correct see sample script in
	%ING_TST%\suites\userenv\tstenv.bat

 B.  EXECUTION PROCESS:
     Before running the FE suites, it is required to execute a script
     to initialize the front-end databases.

     UNIX
       The runfe.sh file initializes and runs the front-end areas
       of Ingres. Initialization creates one or two databases for
       each area of FE.

     WINDOWS
       The runfe.bat performs the same function as runfe.sh on UNIX
 
	1. Initialize the databases.

	   The script can be executed in two ways: a) in its entirety, or b) by
	   facility.

	   a) To initialize ALL databases:

		UNIX:	 % sh $TST_SHELL/runfe.sh init all
		WINDOWS: > %TST_SHELL%\runfe.bat init all

	   b) To initialize selected databases, choose the modules
	      shown between the pipes:

		UNIX:	 % sh $TST_SHELL/runfe.sh init |4gl|abf|image|copy|frs
						       |fstm|qbf|rw|rbf|vision
						       |vifred|genutil

		WINDOWS: > %TST_SHELL%\runfe.bat init |4gl|abf|image|copy|frs
						      |fstm|qbf|rw|rbf|vision
						      |vifred|genutil

	The resulting output of the initialization can be found in
 	$TST_OUTPUT/feinit.out or %TST_OUTPUT%\feinit.out

	The following databases will be created:
		
		abf4gldb
		abfdb1
		abfdb2
		basisdb
		copydb1
		frsdb
		genutildb
		gourmet 
		imagedb
		imagedb2
		qbf1
		rbfdb1
		rwdb
		vifred1

	2. Execute the script.

	   The script can be executed in two ways: a)in its entirety, or
	   b)by facility.

	   a) To execute the script in its entirety:

		UNIX:	 % sh $TST_SHELL/runfe.sh fe all 
		WINDOWS: >%TST_SHELL%\runfe.bat fe all

	   b) To run selected test modules, choose the modules shown
	      between the pipes:

		UNIX:	 % sh $TST_SHELL/runfe.sh fe |4gl|abf|image|copy
						     |frs|fstm|qbf 
						     |rw|rbf|vision
						     |vifred|genutil

		WINDOWS: > %TST_SHELL%\runfe.bat fe |4gl|abf|image|copy
						    |frs|fstm|qbf 
						    |rw|rbf|vision
						    |vifred|genutil

 D.  MONITOR PROGRESS OF FRONT-END TEST:

     Open 3 windows to view the performance of the test:

        1. To execute the commands to run the test suite

        2. Watch the errlog.log file
           (located in II_SYSTEM/ingres/files)
            
		UNIX/WINDOWS: 
		tail -f errlog.log

	3. To monitor the test progress, go to 
	  (located in TST_OUTPUT/fe/local/suitename)

		UNIX/WINDOWS:
            	tail -f [any output file named above in section C.]
